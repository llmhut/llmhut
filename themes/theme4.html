KeyGate — Flagship

Real keys. Real isolation.
Zero proxy overhead.

Most "key management" tools are proxies in disguise. KeyGate uses vendor Admin APIs to create genuine scoped projects and issue actual sk-proj-... keys per developer. Your traffic goes straight to the vendor — KeyGate never touches a request.

Real vendor keys
Not proxy tokens. Developers use authentic sk-proj-... keys that work directly with any SDK, no changes needed.
0ms overhead
Per-developer isolation
Each key lives in its own vendor project or workspace, perfectly isolated. Revoke one dev without affecting anyone else.
Vendor-native isolation
Budget caps that stick
Spend limits enforced at the vendor level — not just alerts, not estimates. A $50 cap means the key stops at $50, guaranteed.
Vendor-enforced
Rate limiting
Per-key RPM limits enforced by the vendor. Protect your shared org quotas from runaway scripts and rogue jobs.
Per-key RPM
Immutable audit log
Every provision, revoke, rotate, and config change is permanently logged with actor, timestamp, and key hint. CISO-ready from day one.
Append-only log
Instant revocation
One API call. The key is deleted at the vendor level before the response returns. Developer offboarded, key dead — simultaneously.

How it works

Zero added latency by design.

KeyGate is only in the loop at key lifecycle time. Inference traffic goes straight to the vendor — KeyGate never sees a prompt.

Step 01
Admin provisions
Admin selects developer + vendor in the dashboard and clicks Provision.
POST /api/v1/keys/provision
Step 02
KeyGate calls vendor API
Creates a scoped project and service account. Stores only the key hint — never the full value.
POST /v1/organization/projects
POST /projects/{"{id}"}/service_accounts
Step 03
Key delivered once
The real key is returned exactly once. Admin shares it via your preferred secure channel.
sk-proj-...7xKm ← shown once only
Step 04
Developer calls vendor directly
Traffic goes straight to OpenAI / Anthropic / Gemini. KeyGate is never in the request path.
openai.api_key = "sk-proj-..."
// 0ms overhead, no proxy
